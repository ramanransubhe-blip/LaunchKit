# Troubleshooting Guide

Fixing common errors, database drops, and webhook connection failures.

---

## Purpose
This document provides a categorized diagnostic reference to resolve common build issues, database connection errors, and webhook pipeline errors in DevLaunchKit.

## Prerequisites
- Running environment locally
- DevLaunchKit CLI doctor command available

## Issue Catalog

### 1. Build and Compilation Issues

#### Error: `Cannot find module '@devlaunchkit/X' or its corresponding type declarations`
- **Cause**: Workspace package symlinks are broken or outdated, or the package was not compiled.
- **Resolution**:
  1. Run `pnpm install` in the root workspace folder to re-register workspace connections.
  2. Run `pnpm build` to compile the package's TS declarations.
  3. If using VS Code, run `TypeScript: Restart TS Server` in the command palette.

#### Error: `ELIFECYCLE pnpm run build exited with code 2`
- **Cause**: Strict TypeScript compilation errors in one or more workspace packages.
- **Resolution**:
  1. Run `pnpm typecheck` to list the exact files and lines that failed types checks.
  2. Fix the type signatures or cast variables to resolve compiler concerns.

---

### 2. Database Connectivity Issues

#### Error: `Database connection timed out / Refused`
- **Cause**: The PostgreSQL docker container is not running or the DATABASE_URL is misconfigured.
- **Resolution**:
  1. Check if the container is running: `docker ps`.
  2. If not running, start it: `docker-compose up -d`.
  3. Verify that no other local PostgreSQL instance is running on port `5432`:
     - Windows: `netstat -ano | findstr 5432`
     - Linux/macOS: `sudo lsof -i :5432`

---

### 3. Webhook Delivery Failures

#### Error: `Webhook signature verification failed (Status 400)`
- **Cause**: The webhook signature token mismatching or missing.
- **Resolution**:
  1. Verify your `STRIPE_WEBHOOK_SECRET` matches the secret generated by your Stripe local CLI session (`whsec_...`).
  2. Ensure you are forwarding webhooks to the correct path: `/api/webhooks/billing`.
  3. Run your tunnel: `stripe listen --forward-to localhost:3000/api/webhooks/billing`.

## Best Practices
- **Run pnpm doctor first**: It checks Node versions, pnpm, env variables, and DB connection, pointing you directly to the root cause of setup errors.
- **Clear cache folders**: When in doubt, clean compiler and workspace artifacts: `pnpm clean` followed by `pnpm install` resolves 90% of local monorepo caching quirks.

## Common Mistakes
- **Modifying packages without rebuilding**: Editing code in a package and expecting Next.js to pick it up immediately, forgetting that ESM compilation is required.
- **Hardcoding connection ports**: Putting direct connections to container endpoints without using environment variable overrides.

---

## Troubleshooting
- **Docker Compose fails to boot**:
  - Run `docker system prune -a` to clean unused images, or check that docker daemon is running on your machine.
- **API keys mismatching**:
  - Run `pnpm doctor` to verify that API tokens do not contain leading or trailing spaces.
