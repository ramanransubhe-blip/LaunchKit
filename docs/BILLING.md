# Billing & Payments Guide

Configuring and managing the provider-agnostic Billing Platform.

---

## Purpose

This document guides you through setting up billing gateways, synchronizing pricing plans, handling webhook events, and implementing subscription lifecycles in DevLaunchKit.

## Prerequisites

- Stripe account credentials (API keys and webhook secrets)
- Dodo Payments sandbox token (if using Dodo Payments as your checkout merchant)

---

## Architecture & Abstraction

DevLaunchKit decouples payment operations via `@devlaunchkit/payments`. By default, Dodo Payments is configured as the active merchant, but you can switch to Stripe by configuring your Stripe credentials.

## Provider Setup

### Dodo Payments Configuration (Default)

Dodo Payments acts as a merchant of record, handling global sales tax, VAT, and invoice compliance automatically.

1. Sign up on [Dodo Payments](https://dodopayments.com) and get your sandbox keys.
2. Add your keys to `.env`:
   ```env
   DODO_PAYMENTS_API_KEY=dodo_sk_...
   DODO_PAYMENTS_WEBHOOK_SECRET=dodo_whsec_...
   ```

### Stripe Configuration

Stripe is the leading payment processor. To active it:

1. Configure your Stripe API keys in `.env`:
   ```env
   STRIPE_API_KEY=sk_test_...
   STRIPE_WEBHOOK_SECRET=whsec_...
   ```
2. Once the `STRIPE_API_KEY` is detected, DevLaunchKit switches its active payment client to the Stripe adapter, rendering Stripe pricing widgets and processing checkout sessions.

## Webhook Ingestion Flow

Webhooks are validated cryptographically inside the `@devlaunchkit/payments` middleware:

```typescript
// apps/web/app/api/webhooks/billing/route.ts
import { payments } from "@devlaunchkit/payments";

export async function POST(req: Request) {
  const body = await req.text();
  const signature = req.headers.get("stripe-signature") || req.headers.get("dodo-signature") || "";

  try {
    const event = await payments.verifyWebhook(body, signature);

    switch (event.type) {
      case "subscription.created":
        await handleSubscriptionCreated(event.data);
        break;
      case "subscription.cancelled":
        await handleSubscriptionCancelled(event.data);
        break;
    }

    return new Response(JSON.stringify({ received: true }), { status: 200 });
  } catch (err: any) {
    return new Response(`Webhook Error: ${err.message}`, { status: 400 });
  }
}
```

## Best Practices

- **Never hardcode product IDs**: Store Stripe / Dodo plan price IDs in the database and pull them dynamically based on user selections.
- **Listen for Webhook Retries**: Payment operations are asynchronous. Always update subscription status via webhook listeners rather than relying on the frontend redirect screen alone.

## Common Mistakes

- **Incorrect Webhook URL**: Pointing Stripe webhooks to `http://localhost:3000` directly without running a tunnel like Stripe CLI (`stripe listen --forward-to localhost:3000/api/webhooks/billing`).
- **Missing Webhook Secret**: Setting the wrong webhook signature key, which will cause all incoming webhook payloads to fail authentication.

---

## Troubleshooting

- **Webhook Signature Verification Fails**:
  - Verify that the webhook signing secret in `.env` matches the secret generated by your Stripe CLI or Stripe dashboard webhook panel.
- **Drizzle Database Out of Sync**:
  - Run the db seed script (`pnpm --filter @devlaunchkit/database db:seed`) to reset subscription plans and sync products catalog definitions.
